# GoDive MVP App Summary

High-level architecture and conventions. For detailed **change history**, see `cursor/change_log.md`. For **coding rules**, see `cursor/rules.md`.

---

## Stack and entry

- **SwiftUI** UI, **SwiftData** persistence, iOS **26** target.
- **`GoDiveMVP/GoDiveMVPApp.swift`:** `@main`; production SwiftData via **`AppModelContainer.production`**. **`AccountSession`** + **`AppSessionRootView`** gate **`ContentView`** behind **Sign in with Apple** (**`SignInView`**). **Launch mock seeding is off** (**`MockDataSeeding.isLaunchSeedingEnabled`** default **`false`**). **MapKit warm-up:** **`MapKitWarmupView`** + **`MapKitWarmup`** run once at launch (hidden **1×1** **`Map`**, brief off-screen **`MKMapView`**) so the first **Explore** / dive map is less janky — skipped under UI tests. **UI tests** (**`-GoDiveUITest`** / env **`GoDiveUITest=1`**): **`GoDiveUITestRootView`** only — no SwiftData init, no MapKit. **Release:** no mock seeding on launch (see **Mock data & testing**).

## App shell and navigation

- **Sign in:** **`SignInView`** — **`WaterBubbleBackground`** with a light full-screen **`surface`** scrim, **`GoDiveLogoPin`** + three-line tagline, native **`SignInWithAppleButton(.continue)`** (scopes **`.fullName`**, **`.email`**); **`AccountSession`** persists **`UserProfile.displayName`** in SwiftData. Apple **`fullName`** (once per Apple ID) or cached name is applied via **`applyDisplayNameFromApple`**; otherwise the profile stays **Diver** with no sign-up name sheet. **Sign out** on **Profile** clears the session pointer (local dives remain for that Apple ID).
- **Root:** `GoDiveMVP/Views/Pages/ContentView.swift` — **`TabView(selection:)`** with iOS 18+ **`Tab(value:)`** (Home, Logbook, Field Guide, Explore). Re-tapping **Logbook** uses the system tab-bar scroll-to-top behavior (iOS 18+), plus a delegate-chaining fallback for the custom logbook list layout. **`tabBarMinimizeBehavior(.onScrollDown)`** (iOS 26 Liquid Glass) only collapses the bar while scrolling — it is not the scroll-to-top API.
- **Tabs:** Home (`LogOverviewView`), Logbook (`LogbookView`), Field Guide (`FieldGuideView`), Explore (`ExploreView`) — all under **`Views/Pages/`** unless noted.
- **Navigation:** Home (and Logbook / Field Guide per current wiring) use **`NavigationStack`** for pushes. Pushed flows that should hide the system tab bar use **`hidesBottomTabBarWhenPushed()`** (e.g. Profile, Settings, Activity upload, Trip Planner, single-dive screen).
- **Secondary destinations:** Pushed headerless flows use **`SecondaryDestinationBackButton`** and related chrome from **`Views/Components/SecondaryDestinationChrome.swift`**.

## Headers and page wrappers

### Top chrome & header strategy

- **Visual:** **`AppStatusBarEdgeScrim`** ( **`AppHeader.swift`** ) — **narrow** fade at the **status bar** only (**`surfaceElevated` → `surfaceGradientTop` → clear**); **GoDive** / **+** / dive tab rows sit on the uninterrupted **`screenBackgroundGradient`** (no full-row chrome tint). **Reduce Transparency** uses a short solid **`surfaceElevated`** band.
- **Layout invariant:** Never reserve header clearance with outer **`padding(.top:)`** on **`ScrollView`** / **`List`** — that keeps rows **below** the bar so nothing composites **under** it. Prefer an **in‑scroll** top **`Color.clear`** spacer (or Logbook’s measured first row) plus **`scrollContentBackground(.hidden)`** where the scroll surface would otherwise occlude the page gradient.
- **Sizing:** **`AppHeaderMetrics.HeightKey`** publishes the height of **`AppHeader`** and of Logbook’s **top chrome row** (same padding as **`AppHeader`**); **`AppTheme.Layout.appHeaderClearanceFallback`** seeds spacer height before the first layout pass.
- **Home & `AppPage`:** Root **`GeometryReader`** passes **`safeAreaInsets.top`** into **`AppHeader`** (**`statusBarSafeAreaTop`**); **`ZStack`** overlay — **GoDive** wordmark **horizontally centered**; **`headerTitleForegroundGradient`**; **`ScrollView`** full height; spacer **inside** scroll axis.
- **Logbook:** **`AppHeaderlessPage`**; **`LogbookTopChrome`** — site search inline with **+**, **`LogbookTopChromeScrim`** (opaque top fade over list, under chrome) + **`HeightKey`** (same padding model as **`AppHeader`**). **`DiveLogbookSiteSearch`** filters by **`resolvedSiteName`**; **Cancel** ends search and dismisses the keyboard; list **`scrollDismissesKeyboard(.interactively)`**. **`GeometryReader`**: top inset = **`safeAreaInsets.top` + chrome `HeightKey`**; bottom scroll padding = **`safeAreaInsets.bottom`** — **`List`** **`ignoresSafeArea(edges: [.top, .bottom])`** so rows scroll under the frosted tab bar; **No matching dives** when a non-empty query filters everything out; swipe delete → single **Delete dive?** modal (no second **Renumber** sheet — see **Settings** for automatic **1…n** renumbering); **Delete** dismisses the modal **instantly**, removes the row via **`optimisticallyRemovedActivityIDs`**, and updates **#** labels on the main thread when auto-renumber is on; heavy duplicate scan + row cache rebuild runs off the main actor via **`LogbookDisplayCacheBuilder`** / **`LogbookCacheRefreshScheduler`**; **`LogbookListSurface`** + paused bubbles avoid redrawing on every SwiftData merge during delete. **`DiveActivityDeletion.Request`** drives delete + optional renumber (**`AppUserSettings.automaticallyRenumberDives`**). **`DiveBackgroundDeletionWorker`** batch-deletes equipment, profile points, buddies, and media by **`diveActivityID`**, nullifies **`diveSite`**, then batch- or object-deletes the dive; video files removed on the worker context. Logbook uses optimistic row hide + **#** relabel when renumber is on (no post-delete full cache rebuild). **`LogbookDiveDeleteProgressOverlay`** until background work + short main-context merge finish. **`zIndex`** for modal above chrome.
- **`ViewSingleActivity`:** **`AppHeaderlessPage`**; back + icon strip in **`overlay`** above map content (not buried under MapKit). Opening a dive builds chart/media derived data off the main actor (**`DiveDerivedDataBuilder`**) and only loads catalog **`DiveSite`** rows when the map needs a name-based coordinate lookup (no always-on **`@Query`**); MapKit mounts immediately on the **map** tab. **Tabs:** **22** pt **`Image(systemName:)`** for map/camera; **`ScubaTankTab`** template (**RGBA**, trimmed) sized with **`templateAssetSize`** to **22** pt height and intrinsic aspect (**`fixedSize()`**) so it aligns with map/camera SF Symbols (**`DiveActivityTabIcon`**). Switching overview tabs resets the embedded sheet to **medium** detent.
- **Other tabs:** Field Guide uses **`AppHeaderlessPage`**; **Explore** — **`ExploreTopChrome`**: map/list toggle **leading**, Trip Planner **trailing** (same row alignment as **`AppHeader`** back / Logbook **+**); full-bleed **`ExploreCatalogMapView`** with system **red `MKMarkerAnnotationView`** markers for catalog **`DiveSite`** rows with coordinates, or logbook-style **`ExploreDiveSiteRow`** list of all sites (tap → **`ExploreDiveSiteDetailSheet`**). Single-dive **`DiveLocationMapView`** uses the same system red marker with **`titleVisibility = .visible`** and locale-formatted coordinates via **`mapMarkerCoordinateTitle`**; site name stays on the overview sheet header. Stack-level **`navigationInteractivePopGestureForHiddenNavBar()`** where needed. **`.cursor/rules/headerless-pages-default.mdc`:** new pages default **headerless**; optional **`AppStatusBarEdgeScrim`** + **`AppHeaderMetrics`** like Logbook when scroll-under top controls are needed. Pushed **secondary** flows use **`SecondaryDestinationChrome`** / **`SecondaryDestinationBackButton`** where applicable.
- **`AppPage`:** Settings, Trip Planner, Add activity, etc. — same **`AppHeader`** **`ZStack`** + measured inset as Home; hides the **system** navigation bar for a consistent look. **`scrollContentUnderHeader: true`** (Equipment Locker, Certifications) uses **`AppScrollUnderHeaderList`** / **`AppScrollUnderHeaderEmptyState`** (read insets inside **`AppPage`** content) + **`LogbookTopChromeScrim`**, matching Logbook / Explore list top spacing.
- **`AppHeaderlessPage`:** Hides the default nav bar; applies **`screenBackgroundGradient`** with **`ignoresSafeArea()`** for full-bleed chrome (see **Theme and chrome**).
- **Swipe back (hidden system bar):** **`NavigationInteractivePopGestureEnabler.swift`** — **`UIViewController`** anchor + **`UINavigationController`** **`@retroactive UIGestureRecognizerDelegate`** (pop + simultaneous scroll pan); optional foreground **nav** walk when the anchor has no **`navigationController`**; **`goDiveLeadingEdgeSwipePopOverlay`** (narrow leading strip + optional **`onWillDismiss`** + **`dismiss()`**). Applied on **`AppPage`**, **`AppHeaderlessPage`** (optional **`leadingEdgePopOnWillDismiss`**), and outer **`NavigationStack`**s where needed. **`GoDiveMVPTests`:** UIKit navigation tests are **`@MainActor`**.

## Theme and chrome

- **`Views/Components/AppTheme.swift`:** semantic colors (light/dark), spacing, typography tokens — prefer these over one-off RGB in views. **Full-screen backgrounds** use **`screenBackgroundGradient`** (**`surfaceGradientTop`** → **`surfaceGradientBottom`**). **Status-only** top fade lives on **`AppStatusBarEdgeScrim`** in **`AppHeader.swift`** (not a second full-row gradient token). **`AppTheme.Layout.appHeaderClearanceFallback`** seeds header / Logbook spacer height before **`AppHeaderMetrics.HeightKey`** first fires. **Sheets:** **`AppTheme.Sheet`** corner radius, **`contentTopSpacing`** (24pt below grabber), and material opacity (**0.64** modals; **0.82** embedded dive overview on **map** / **tank**); **`appSheetPresentationChrome()`** / **`appSheetContentTopSpacing()`** on every **`.sheet`** (**`AppSheetPresentation.swift`**, **`.cursor/rules/swiftui-sheet-standard.mdc`**).
- **Launch:** **`LaunchScreen.storyboard`** — **one** inline **light** sRGB background ( **`surfaceGradientBottom`** **light** ), **`GoDiveLogoPin`**, centered **GoDive** label (**`accent`** **light** inline). No catalog **`name=`** on the storyboard (reliable **`ibtool`**). **`LaunchScreen*`** colorsets in **Assets** document gradient/title stops for reference; cold start does **not** switch with Dark Mode until SwiftUI runs.

Canonical summary vs time-series split:

| Model | Role |
|--------|------|
| **`UserProfile`** | Local account: **`appleUserIdentifier`**, **`displayName`**, optional **`profilePhoto`** (**`Data?`**), optional **`danInsuranceNumber`** (DAN), sign-in timestamps; **`@Relationship`** to **`DiveActivity`**, **`EquipmentItem`**, and **`Certification`**. **Profile** shows circular avatar (**`ProfileAvatarEditor`**) with **`accentDeep`** ring above name. |
| **`DiveActivity`** | Dive summary: **`source`** (**`DiveSource`**: Garmin / MacDive / manual), **`startTime`** (UTC instant), optional **`timeZoneOffsetSeconds`** (UDDF site **`geography/timezone`**; else **`DiveGeographicTimeZoneLookup`** from entry/site GPS + **`startTime`** for DST; naive MacDive **`datetime`** as **UTC**; **`Z`/`±HH:MM`**; FIT **`Activity`** delta), depths (**m** in store), temps (**°C**), ascent (**m/s**), **`siteName`** / **`locationName`**, import GPS **`entryCoordinate`**, optional **`diveSite`** + **`diveSiteID`** (catalog link; map uses **`siteCoordinate`** when linked), notes, manual **Conditions** / **Operator** log, optional **tank** fields, **`ownerProfileID`** + **`owner`**, optional **`diveNumber`**, **`diveNumberExplicitlyNone`**; relationships to buddies, **`mediaPhotos`**, profile points, and optional **`equipmentList`**. **UI** formats via **`DiveQuantityFormatting`** + **`diveDisplayUnitSystem`**; dates/times via **`DiveActivityTimePresentation`**. |
| **`DiveProfilePoint`** | Time-series samples per dive (depth **m**, temp **°C**, ascent **m/s**, NDL, time-to-surface, optional **`tankPressurePSI`**). Chart and labels use the same display-units environment as **`DiveActivity`**. |
| **`DiveBuddyTag`** | Buddy name tags linked to a dive. |
| **`DiveMediaPhoto`** | Post-dive **image** (inline **`mediaData`**) or **video** (file under **`DiveMediaFileStore`**, **`mediaKind`**, **`sortOrder`**, optional **`capturedAt`**) on **`DiveActivity.mediaPhotos`**. Import reads EXIF / IPTC / GPS / file / AV metadata, then **`PHAsset`** dates via **`PhotosPicker`** + **`photoLibrary: .shared()`** (matches Photos app for library videos). |
| **`DiveSite`** | Catalog site: **`siteName`**, optional place hierarchy (**`country`**, **`region`**, **`bodyOfWater`** — broadest to narrowest, empty when unset), optional **`latCoords`** / **`longCoords`**, **`siteTags`**, **`siteRating`**; inverse **`diveActivities`**. **`DiveActivitySiteAssociation`** links on import (GPS match, then name). **`DiveSiteReviewIndicator`** when import **`siteName`** ≠ linked catalog name (no dedicated UI yet). |
| **`EquipmentItem`** | Gear inventory: manufacturer, model, **type**, **`isRetired`**, **`autoAdd`** (auto-links on new dive import via **`DiveActivityEquipmentAssociation`**), **`divesUsedOn`** (computed dive IDs from entries), purchase/service fields, optional **`equipmentPhoto`**; **`EquipmentItemOwnership`**. **Equipment Locker** on Home stack (Profile → locker → detail; scroll-under-header list, add sheet, swipe delete) → **`ViewEquipmentDetails`** (**dives used on** in **Status**) + **Edit** sheet. **Single-dive Tank tab** — **Equipment** chevron row opens **`DiveActivityAddEquipmentSheet`** to link locker gear. |

Add/edit: segmented **Type** picker. **`CertificationOwnership`** / **`CertificationDeletion`**. |

## Data storage strategy

- **Single source of truth:** All durable dive, account, catalog, equipment, and certification data lives in **SwiftData** via one **`ModelContainer`** (**`AppSwiftDataSchema`**) in **`GoDiveMVPApp`** (default **on-disk** store). **`@Model`** types include **`UserProfile`**, **`DiveActivity`**, **`DiveActivityEquipmentList`**, **`DiveEquipmentEntry`**, **`DiveProfilePoint`**, **`DiveBuddyTag`**, **`DiveMediaPhoto`**, **`DiveSite`**, **`EquipmentItem`**, **`Certification`**; there is **no second database** and **no archive of raw import file bytes** (**.fit** / **.uddf**) after import.
- **User preferences (`UserDefaults` / `AppUserSettings`):** **`automaticallyRenumberDives`** can rewrite persisted **`diveNumber`** after import, seed, or delete. **`useImperialDisplayUnits`** only drives **`EnvironmentValues.diveDisplayUnitSystem`** and **`DiveQuantityFormatting`** for on-screen strings; SwiftData fields stay in canonical **m** / **°C** / **psi** (no unit migration on toggle). **`defaultTankSize`** (**`DefaultTankSize`**) sets rated cylinder **cu ft** + **aluminum** / **steel** for new imports and gas rows when a dive has no material / volume from file.
- **Dives:** Each saved dive is a **`DiveActivity`** row owned by the signed-in **`UserProfile`** (**`DiveActivityOwnership`** on import / sign-in; legacy unowned rows are **claimed** on next sign-in). The **Logbook** tab lists dives for **`AccountSession.currentProfile`** only (newest first). **`DiveActivityDiveNumbering`** owns chained **file-import** **`diveNumber`** assignment (**`nextChainedDiveNumberForNewImport`**), **`backfillMissingDiveNumbers`** (skips **`diveNumberExplicitlyNone`**), **`renumberAllChronologically`** (full **1…n** by **`startTime`**, skips **`diveNumberExplicitlyNone`** / logbook **`-`**), **`applyAutomaticSequentialRenumberIfNeeded`** (calls **`renumberAllChronologically`** when **`AppUserSettings.automaticallyRenumberDives`** is **on** — after import, seed, and after delete via **`DiveActivityDeletion`**), and **`renumberDivesNewerThanDeleted`** (tail-only; also skips **`-`** dives; tests / internal). **`DiveActivityDeletion`** uses **`DiveBackgroundDeletionWorker`** (**`@ModelActor`**) with SwiftData **batch** **`delete(model:where:)`** for profile points / buddies / media / equipment, then the dive row; **`DiveSiteCatalogMaintenance.deleteSiteIfOrphaned`** for the linked site only; tail renumber via **`DiveBackgroundRenumberingWorker`** (**`startTime`** predicates, not a full-log fetch).
- **Ingress vs storage:**  
  - **JSON fixtures (opt-in Debug):** Bundled **`MockData/*.json`** via **`MockDataSeeder`** when **`MockDataSeeding.isLaunchSeedingEnabled`** is **`true`** (off by default).  
  - **File import (Logbook → Add activity):** **`.fit`** — read via **`FitDiveFileImport.readFitFileData`**, **`FitDiveFileDecoder`**, **`importFitData`** → **`DiveFileImportOutcome`** (**`userMessage`** + **`primaryInsertedDiveId`**). **`.uddf`** — **`UddfDiveFileImport.readUddfFileData`**, **`UddfDiveFileDecoder.buildDiveActivities`** (may return **multiple** dives per file), **`importUddfData`** (same outcome type; multi-dive files set **`primaryInsertedDiveId`** to the **newest** inserted dive by **`startTime`**, **`id`** tie-break). Before insert, **`DiveActivityDuplicateMatcher`** blocks or skips rows that match the log (**same `sourceDiveId`**, or cross-format **fingerprint**: start ±2 min, max depth ±0.6 m, bottom/duration tolerance — e.g. same dive from **FIT** and **UDDF**). **Logbook** shows **Possible duplicate** on matching pairs. Both paths use security-scoped **`Data`**, then **`DiveActivityDiveNumbering.assignNextDiveNumberChainedAfterNewest`** per dive (insert **oldest-first** for multi-dive UDDF so chained **#** matches chronology); **`insert`**; **`DiveActivityEquipmentAssociation.applyAutoAdd`** links owner gear with **`autoAdd`** (not retired); **`save`**. When **`AppUserSettings.automaticallyRenumberDives`** is **on**, **`applyAutomaticSequentialRenumberIfNeeded`** runs so **`diveNumber`** is **1…n** by date. Only canonical fields are written; **source files are not retained**. **`rawImportVersion`** / **`sourceDiveId`** record provenance (**FIT:** **`FileIdMesg`**; **UDDF:** generator string + **`<dive id>`**). **FIT** also fills **`bottomTimeSeconds`**, **`surfaceIntervalSeconds`**, **`waterTempAvgCelsius`** / **`waterTempMaxCelsius`** / **`waterTempMinCelsius`** from **`DiveSummaryMesg`** + session/record temps (**`DiveImportWaterTemperatureSummary`**); **UDDF** sets **`bottomTimeSeconds`** from **`diveduration`**, **`waterTempMinCelsius`** from **`lowesttemperature`** + waypoint min. **`DiveProfilePoint`**: **FIT** maps **`ndl_time`**, **`time_to_surface`**, **`ascent_rate`** when present; **FIT** / **UDDF** may set **`tankPressurePSI`** when the source includes per-sample cylinder pressure. **FIT** tank summary / updates → dive-level **`tankPressure*PSI`**, **`tankVolumeDescription`** (L + **~ft³** from **`TankSummary.volume_used`**); **UDDF** **`tankdata`** / waypoint **`tankpressure`**. After decode, **`DiveSACRMVCalculation`** sets **`avgSAC`** (**psi/min**) and **`avgRMV`** (**L/min**) when pressures, time, and depth allow; **RMV** uses rated tank **L** ÷ working **psi** (UDDF) or **FIT** **`volume_used` ÷ time** when no rated size. **FIT** entry **`DiveCoordinate`** prefers **`SessionMesg`** **`start_position_*`** (then first **`RecordMesg`** fix); **FIT** rejects **>1** diving session or **>2** tank sensor ids (**`FitDecodeError`**).
- **Catalog:** **`DiveSite`** rows (fixture-seeded **`divesites_sample`** when launch seeding is on) with optional **`country`**, **`region`**, **`bodyOfWater`**. On import, **`DiveActivitySiteAssociation`** links by exact import **`siteName`** when present (never a different nearby catalog site); duplicate catalog names disambiguate by GPS within that name set only; GPS match runs only when the dive has no import site name; unmatched import names create a new **`DiveSite`** (single **FIT** / **UDDF** always; bulk UDDF when **Create dive sites** is on). Import GPS stays on **`entryCoordinate`**; map pin uses linked **`siteCoordinate`** when present. Unlinked dives with location hints get a map-tab prompt to **`DiveSiteAddSheet`** (Place + map/coordinates; **info** control after dismiss). Import **`locationName`** prefills **region** / **country** on the add/edit sheet (**`DiveImportedLocationParsing`**: before/after first comma).
- **Not implemented yet:** Storing original FIT or JSON attachments, CloudKit / multi-device sync, export, or merge/delete-duplicate UX beyond **import block** + **Possible duplicate** logbook hint (**`DiveActivityDuplicateMatcher`**).

## Dive file import (Garmin FIT + UDDF)

- **FIT — dependency:** Swift package **[FITSwiftSDK](https://github.com/garmin/fit-swift-sdk)** (v21.x+) on **`GoDiveMVP`**; **`Package.resolved`** in **`swiftpm`** shareddata.
- **FIT — decode + map:** **`Data/Import/FitDiveFileDecoder.swift`** — **`Decoder`** + **`FitListener`**; **`SessionMesg`** with **`Sport.diving`** (exactly **one** diving session per file); **`DiveActivity`** (**`source: .garminMK3`**, **`rawImportVersion`**); **`DiveSummaryMesg`** (session-linked) for **`bottomTimeSeconds`** / **`surfaceIntervalSeconds`**; **`DiveImportWaterTemperatureSummary`** for **`waterTemp*Celsius`**; **`DiveProfilePoint`** from **`RecordMesg`** (depth, time, temp, NDL, time-to-surface, instantaneous **ascent rate** m/s, optional per-sample **`tankPressurePSI`** from **`TankUpdateMesg`** time-alignment, optional **`heartRateBPM`**, **`po2Bars`**, **`n2Load`**, **`cnsLoad`** when present in the file); **entry GPS** from **`SessionMesg.start_position_*`** when set, else first **`RecordMesg`** position (invalid semicircle **`0x7FFFFFFF`** skipped); **tank** dive-level fields from **`TankSummaryMesg`** / **`TankUpdateMesg`** via **`FitTankFieldImport`** (bar→psi; **≤2** tank sensors; **`>2`** sensors or **>1** diving session → **`FitDecodeError`**); **gas** from **`DiveGasMesg`** (**`oxygenMix`** %, **`gasType`** **Air** / **Nitrox** via **`DiveGasMixImport`**); **`sourceDiveId`** from **`FileIdMesg`**. **`FitDecodeError`** for invalid FIT.
- **UDDF — decode + map:** **`Data/Import/UddfDiveFileDecoder.swift`** — **`XMLParser`** for **`http://www.streit.cc/uddf/3.2/`** (e.g. **MacDive** export); **`DiveActivity`** (**`source: .macDive`**), optional **tank** fields from **`tankdata`** (Pa→psi, m³ volume string, optional **`tankmaterial`**), **gas** from **`gasdefinitions/mix`** (**`<o2>`** → **`oxygenMix`** %) via **`tankdata/link`**, **`bottomTimeSeconds`** from **`diveduration`**, **`waterTempMinCelsius`** from **`lowesttemperature`** + waypoint min, profile from **`samples/waypoint`** (including waypoint **`tankpressure`** → **`tankPressurePSI`**), buddies from **`diver/buddy`** + links, site from **`divesite/site`**; **`UddfDecodeError`** for invalid / empty / no dives. Unmapped UDDF fields are listed in **`cursor/todo.md`** (UDDF section).
- **UI:** **Logbook** **`plus`** → **`ActivityUploadView`** — **File upload** (**`.fit` / `.uddf`**), **Bulk upload UDDF dives** (multi-dive **`.uddf`** + live **`N of M`** scrim + completion alert), **Manual entry**; **`DiveFileImportOutcome.didSucceed`** / **`DiveFileImportSuccess`**. SwiftUI **`.fileImporter`** (do not clear **`isPresented`** in **`onDisappear`**); **`DiveFileImporterPresentation`** for UTTypes + picker cancel. **Bulk UDDF** imports **dives only** (activity + profile/tank/site/buddies from UDDF) — **not** MacDive media, equipment locker, certifications, or other export data yet. **Settings** — import hint + **Default tank** + renumber + units.

## Notable screens (MVP)

- **`LogbookView`:** **`NavigationStack(path:)`** with a private **`Route`** enum (**`addActivity`** / **`diveDetail(UUID)`**). **`@Query`** all **`DiveActivity`** (**newest first:** **`startTime`** descending, then **`id`** ascending for ties); **`visibleActivities`** filters **`optimisticallyRemovedActivityIDs`** so the row vanishes on **Delete** immediately; **`LogbookDisplayCacheBuilder`** (fed by **`LogbookActivitySnapshot`**) builds **`Equatable`** row data and duplicate IDs off the main actor, with debounced **`LogbookCacheRefreshScheduler`** updates after store changes (longer debounce right after delete). Site search filters rows only; **#** labels still use the full visible logbook for numbering. **Top:** **`LogbookTopChrome`** — **`LogbookSiteSearchField`** + **Cancel** (when focused) + **+** on one row. **`GeometryReader`**: inset = **`safeAreaInsets.top` + `AppHeaderMetrics`** (**`List`** **`ignoresSafeArea(edges: .top)`**, hidden list background). When **Settings → Automatically renumber dives** is on, logbook **#** is derived from chronological order among numbered dives only (hidden **`-`** rows stay **`-`** and do not consume a **#** slot; not live **`diveNumber`** during background persist). **Hide dive number in logbook** (**`diveNumberExplicitlyNone`**) shows **`-`** on the row (including when auto-renumber is on); row-cache signature includes hide + number so returning from a dive detail refresh works. **`LogbookActivityRow`** may show **Possible duplicate** when **`DiveActivityDuplicateMatcher`** finds a fingerprint or **`sourceDiveId`** match. **+** and empty-state CTA push **`addActivity`**. Swipe **Delete** → **Delete dive?** modal; **Delete** shows **`LogbookDiveDeleteProgressOverlay`** immediately, optimistic row hide + optional **#** relabel, optional detail pop, then **`DiveActivityDeletion.delete`** on a **`.utility`** **`Task`** (batch child deletes, tail renumber when Settings allow, ~500ms main-context merge cap). **`List`** uses **`animation(nil, value: visibleActivities.count)`**. **`navigationDestination`:** **`ActivityUploadView`** with **`onSuccessfulImport`** (pop upload, push **`ViewSingleActivity`** for **`primaryInsertedDiveId`**). Empty state → **`NavigationLink(value: .addActivity)`**.
- **`SettingsView`:** **`AppPage`** titled **Settings**; compact rows (**`SettingsToggleRow`**, **`SettingsPickerRow`**) — title + control + **info** alert for full copy (**`SettingsPresentation`**); **Imperial units**, **Default tank** (**`DefaultTankSize`**: **AL80**, **AL63**, **ST100**, **ST120**), **Automatically renumber dives**; enabling renumber runs **`renumberAllChronologically`** once; import/seed/delete honor **`AppUserSettings`** (see **Data storage**). Imperial toggle only affects on-screen formatting (**`DiveQuantityFormatting`**), not persisted **`DiveActivity`** fields.
- **`ActivityUploadView`:** **`AppPage`** titled **Add activity**; **`onSuccessfulImport(UUID)`** / **`onBulkImportComplete`**. **VStack:** **File upload** (**.uddf or .fit**), **Manual entry**, compact **Bulk UDDF** row (**.uddf with multiple dive records**). **One** SwiftUI **`.fileImporter`** (**`DiveFileImporterPresentation.PickerMode`** for single vs bulk UTTypes); **`presentFileImporter`** defers presentation one run loop; bulk opens after options sheet **`onDismiss`**; **`onDisappear`** never clears picker **`isPresented`** mid-flight. **`.large`** bulk options sheet: MacDive **Settings > Export these dives to UDDF** copy, **Create dive sites from import** toggle (**`AppUserSettings.bulkUddfCreateDiveSitesKey`**, default on), then file picker. Bulk scrim: per-dive **`N of M`**; **Import complete** alert (**dives imported**, **duplicates**, **dive sites created**). **Scope:** bulk path writes **`DiveActivity`** (+ UDDF-mapped fields) only — **no** MacDive photos/videos, locker gear, or other export payloads yet. **`DiveActivityDuplicateMatcher`** (cross-format FIT/UDDF). **`UddfDiveFileImport`** fast persist + site creation count. **`DiveActivitySiteAssociation`**: unique exact **`siteName`** before GPS. **UDDF** **`divenumber` `0`** → **`-`** in logbook.
- **`ViewSingleActivity`:** **Map-first** from **Logbook** — **`DiveLocationMapView`** / **`DiveTankOverviewHeroView`** behind **`DiveActivityOverviewEmbeddedPanel`** (three detents **~20% / ~50% / ~85%**). **Map** panel: site header + **`DiveActivityEditableSectionsView`** (**map** tab) — grouped editable rows (**`DiveActivityEditableRow`**, chevron when tappable; **Signature** shows a drawing preview when saved); Dive section shows **Start (dive time)**, read-only **Start (UTC)** (ISO **`Z`**) and **Timezone offset** when import stored **`timeZoneOffsetSeconds`**; **`DiveActivityFieldEditSheet`** per field; **`DiveActivityBuddiesEditSheet`**; **Linked catalog site** opens **`DiveSiteAddSheet`**. **Tank** hero (**minimized**): **`DiveDepthProfileOverlayChart`** (depth + PSI). **Portrait:** chart + small cylinder + PSI-used summary (no media markers) + animated **rotate phone** hint above the sheet. **Landscape** (phone sideways): full-width **`DiveDepthProfileOverlayChart`** with **square media thumbnails** when **`capturedAt`** is on the axis (pinch to zoom — markers grow up to **2.5×** with zoom; **two-finger** pan when zoomed; **one-finger** hold-and-drag scrub for depth/PSI; double-tap reset; tap thumbnail → **`DiveDepthProfileMediaPreviewSheet`** at **large** detent); cylinder, PSI-used text, and the **overview sheet** hidden. Landscape flips disable implicit orientation animations on the hero/panel, defer thumbnail markers until rotation settles, and rasterize profile polylines; precomputed depth/pressure/marker arrays are reused per render pass; profile-series sorted overloads and media plotting now reuse one dive time axis per marker/context batch (instead of recomputing per media item). Detent transitions on **map/tank/media** reuse cached derived dive data (**sorted profile/media arrays, depth+pressure samples, markers, capture contexts, lookup maps, gas stats**) refreshed only when dive/profile/media inputs change. **Tank** panel: editable gas/cylinder/consumption + **Equipment** chevron row (depth + PSI profile lives in the hero **`DiveDepthProfileOverlayChart`** only). **Media** tab: full-bleed pager (photos + inline **`VideoPlayer`**) + sheet **+** picker (images and videos, up to 20 per add) (**`DiveActivityAddEquipmentSheet`**); media selection sync now uses a lightweight signature (**count + first/last ID**) instead of joining all IDs each render. Nearly all **`DiveActivity`** scalar fields are editable at **medium** detent (unit-aware via **`DiveActivityFieldValueParsing`**); read-only: profile sample counts, record ID, owner. Map site prompt, map teardown, tank hero pressure animation unchanged. **`DiveActivityDetailsPresentation`** remains for tests / legacy. Top chrome: back + icon tabs in **`overlay`**; **`view_single_activity`**.

## Not yet built / placeholders

These gaps are intentional for an MVP shell; **`cursor/rules.md`** also lists larger deferrals (Garmin import, sync, media, etc.).

- **Tabs with almost no content:** **Home** and **Field Guide** use **`AppComingSoonPlaceholder`** (dashboard / species guides) + bubbles. **Explore** — map or list of **`DiveSite`** catalog + Trip Planner link. Pins appear when the store has seeded or imported catalog rows with coordinates (launch seeding off by default). **Profile** (from Home): dark scrim; top bar **back**, **⋮** edit → **`ProfileEditSheet`**, **Settings** (gear); circular photo with **crop-before-save** (pan/zoom clamped to image); centered **`displayName`**, optional **DAN**, primary cert, owned **dive count**; square **Certifications** / **Equipment Locker** tiles show owned counts (**`ProfilePresentation`**); **Sign out**. **Home** header **`ProfileAvatarView`** (**`accentDeep`** ring; saved photo or **`person.circle.fill`**). **Logbook** lists the signed-in user’s dives (see **Notable screens**).
- **Home is not a real log:** The **Home** tab is a placeholder shell (header, coming-soon copy, profile link); use **Logbook** to open dives.
- **Single-dive screen:** Icon tabs (**map**, tank, **Photos**) on the overview; all three share an **embedded** bottom panel (**`DiveActivityOverviewEmbeddedPanel`**, three detents + grabber) in the same **`NavigationStack`** destination so back navigation moves the panel with the page (not a lingering modal **`.sheet`**). **Map** / **tank** at **minimized** still use a lightweight **collapsed summary** overlay; **Media** keeps one mounted **`photosOverviewPanelContent`** at all detents (**`showsPanelContentWhenMinimized`**, scroll disabled in the compact band) so the carousel is not torn down when dragging between **minimized** and **medium**. **Media** tab: swipeable **`DiveActivityMediaBackgroundView`** hero pager (**No media added** when empty); **minimized** and **medium** sheet show **`DiveActivityMediaCarouselView`** (oldest **`capturedAt`** left → newest right; tap thumbnail to jump the hero; minimized taps do not expand the sheet); **medium** also shows **Media** header, position label, and **Captured** time; video carousel frames reuse **`DiveActivityVideoThumbnailCache`**; videos play **muted** and **loop** on the visible page (restart from the beginning when swiping or tapping the carousel to a new clip; **hold still** ~0.2s on the video to pause until release (small finger movement cancels so horizontal swipes page) while the tab is selected and the sheet is **minimized** or **medium** (paused at **large** or on other tabs); **`DiveMutedVideoAudioSession`** (**`.ambient`** + mix-with-others) so playback does not stop music/podcasts from other apps; at **minimized** the hero shows capture date/time + **Captured at** depth / minutes-into-dive subtitle (same as profile preview); sheet **+** **`PhotosPicker`** → **`DiveActivityMediaBatchImport`** + **`DiveMediaImportProgressOverlay`**. On back / leading-edge pop, live **`DiveLocationMapView`** swaps to **`DiveOverviewMapTeardownPlaceholder`** before the pop animation. **Details** and **More** (debug dump) remain in source for later.
- **Data model gaps:** **`DiveActivity`** still stores **`siteName`**, **`locationName`**, and **`DiveCoordinate`** as free text / embedded values. **`DiveSite`** exists as a **catalog** table and supports **coordinate matching** / **`needsReview`** logic, but there is **no `@Relationship`** from a dive to a chosen **`DiveSite`** yet, no user “accept suggestion” persistence, and no general **location** model beyond strings on the dive.
- **Production data path:** **`.fit`** and **`.uddf`** via **Logbook** → **Add activity**; mock JSON seeding is opt-in only (**`MockDataSeeding.isLaunchSeedingEnabled`**). **Duplicate detection** blocks re-import of the same dive (including cross-format); no merge/delete-duplicate UX yet. Full-field parity with every exporter is still incomplete (**`cursor/todo.md`**).
- **UI tests:** **`GoDiveMVPUITests`** is a minimal template, not a regression suite.

## Mock data & testing

- **Policy:** Non-trivial **`GoDiveMVP/`** changes should add or update tests in **`GoDiveMVPTests`** or **`GoDiveMVPUITests`** (see **`.cursor/rules/code-changes-require-tests.mdc`** and **`cursor/rules.md`**).
- **Debug builds:** **`MockDataSeeder`** available but **disabled** at launch by default; set **`MockDataSeeding.isLaunchSeedingEnabled`** to load bundled JSON. **Release:** never seeds on launch.
- **Unit tests:** **`GoDiveMVPTests`** — Swift Testing, logic-focused: matchers, **`.fit`** / **`.uddf`** import decoders + outcomes, **`DiveActivityDuplicateMatcher`**, **`DiveSACRMVCalculation_*`**, depth-profile math, **`diveLocationMapPresentation_*`** (incl. **`adjustedMapCenter`** with top + bottom obstructions), **`diveMapCoordinateResolver_*`**, **`diveActivityOverviewDetent_*`**, **`diveActivityOverviewPanelMetrics_*`** (scroll snap, detent helpers), tank tab (**`diveActivityTankPanelSummary_*`**), tab icons, **`DiveQuantityFormatting`**, **`DiveActivityDeletion`**, **`AppUserSettings`**, **`FitTankFieldImport_*`**, etc.
- **UI tests:** **`GoDiveMVPUITests`** — **`XCUIApplication.goDiveForUITesting()`** launches **`GoDiveUITestRootView`** (stable AX shell, not production **`ContentView`**). **`testLaunch`**: **`GoDive.UITest.Root`**, Home tab, Profile. Scheme: test bundles **`parallelizable = "NO"`**; **`runsForEachTargetApplicationUIConfiguration = false`**.
- **Manual:** **⌘R** in Debug on simulator/device with fixtures; previews where present.
