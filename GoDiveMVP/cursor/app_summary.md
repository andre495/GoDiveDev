High-level architecture and conventions. For detailed **change history**, see `cursor/change_log.md`. For **coding rules**, see `cursor/rules.md`.

## Stack and entry

- **SwiftUI** UI, **SwiftData** persistence, iOS **26** target.
- **Maps:** **MapKit** when no Google key is configured. **Google Maps SDK** when **`Config/GoogleMapsSecrets.plist`** contains a valid **`APIKey`** (**`GoDiveMapEngine`**); **Explore**, dive overview, and site coordinate picker switch automatically. Optional **`MapID`** for hybrid POI styling. Launch arg **`-GoDiveMapEngineGoogle`** still forces Google without secrets (tooling).
- **`GoDiveMVP/GoDiveMVPApp.swift`:** `@main`; **`AppNetworkConnectivityMonitor`** (**`NWPathMonitor`**) at launch — offline uses local PhotoKit previews only (no iCloud / full-res fetch); dive media shows **`OfflineMediaUnavailableIndicator`** instead of retry error chrome. Production SwiftData via **`AppModelContainer.loadProduction()`** (on-disk **`ModelContainer`** init on a background thread, then **`.modelContainer`** on the root — avoids main-thread I/O hang at launch). **`AppLaunchOverlay`** bridges the system **`LaunchScreen.storyboard`** only while **`AccountSession`** restores the local profile (**`AppSessionBootstrapPresentation`**); Home featured media warms on **`LogOverviewView`** (slide **0** first, slides **1…n** deferred). Carousel excludes library videos **> 30 s**. **`AccountSession`** + **`AppSessionRootView`** gate **`ContentView`** behind **Sign in with Apple** (**`SignInView`**). **Launch mock seeding is off** (**`MockDataSeeding.isLaunchSeedingEnabled`** default **`false`**). **Launch maintenance:** **`AppLaunchMaintenance`** — dive # backfill, buddy legacy migration, marine catalog seed on a background **`ModelContext`** (not on the main thread at attach). **MapKit warm-up:** **`MapKitWarmup`** runs once ~**400 ms** after root attach (off-screen **`MKMapView`** only — no launch **`Map`** in the tree) so the first **Explore** / dive map is less janky — skipped under UI tests. **Google Maps warm-up:** **`GoogleMapsWarmup`** (hidden **hybrid** **`GMSMapView`**, same deferral) when **`GoogleMapsBootstrap.shouldWarmUpAtLaunch`**. **UI tests** (**`-GoDiveUITest`** / env **`GoDiveUITest=1`**): **`GoDiveUITestRootView`** only — no SwiftData init, no MapKit. **Release:** no mock seeding on launch (see **Mock data & testing**).

### Google Maps — auth model (no user OAuth today)

The **Maps SDK for iOS** authenticates with a **server-side API key** bundled in the app (see **Secrets / setup** above). It does **not** sign the user into a Google account or send dive log data to Google for basic tile + marker display.

**Not required for current spike:** Google **OAuth consent screen** or “share data with Google Maps” user flow.

**Revisit before adding:** Google **Places** (autocomplete / search), **Routes**, **Geocoding** as a billed Google REST API, **Google Sign-In**, or any API that reads a user’s Google account or sends user-generated dive content to Google backends — those may need a privacy-policy update, OAuth consent configuration, and/or additional API enablement. Track in **`cursor/todo.md`** → **Google Maps / Google Cloud**.

### Fishial.AI — optional cloud fish identification

GoDive depends on **[Fishial.AI](https://fishial.ai/)**’s hosted **Recognition API v2** for the manual **Identify fish** flow on dive media (sparkles on the **Marine life** tag sheet toolbar). The feature is **off unless configured**: **`FishialSecretsBootstrap`** loads **`ClientID`** / **`ClientSecret`** from gitignored **`GoDiveMVP/Config/FishialSecrets.plist`** (free developer keys from [portal.fishial.ai](https://portal.fishial.ai)).

**Selection UX:** photos open the crop editor on a fast local preview (**`.fastFormat`**, **1024** edge); videos scrub on the lighter **`.homeCarousel`** stream with duration from **`PHAsset`** metadata — full iCloud originals load only when the user taps **Identify** / **Continue**. **What leaves the device:** one **JPEG still** per identify (user-cropped full-quality photo export or cropped scrubbed video frame) via **`POST /v2/recognize`**, plus optional dive coordinates in the **`Fishial-Location-Lat-Lon`** header when the associated **`DiveActivity`** has a resolvable map coordinate (linked site, entry GPS, or catalog name match). **No automatic batching** — the user must tap **Identify** after framing the fish in the crop editor (videos: scrub → **Continue** → crop → **Identify**).

**What stays local:** PhotoKit originals, the full dive log, and Field Guide catalog data. Fishial returns species candidates only; GoDive **fuzzy-matches** those scientific names against the bundled **`MarineLife`** catalog (**`FishialMarineLifeCatalogMatching`**) and shows catalog thumbnails in the review sheet. When the user confirms a match (single result) or picks one from multiple, GoDive creates a **`SightingInstance`** tag on that catalog species (**`MarineLifeSightingRecorder.tagSpecies`**) and saves the catalog scientific name on **`DiveMediaPhoto.fishialConfirmedSpeciesName`**; the matching species oval at **medium** / **large** shows an in-chip **sparkles** badge.

**Operational notes:** requires network and a signed-in profile to save catalog tags; subject to Fishial account credits, API availability, and their [API terms / privacy policy](https://docs.fishial.ai/api). Client code: **`FishialAPIClient`**, **`FishialObservationLocation`**, **`DiveMediaFishialIdentification`**, **`FishialMarineLifeCatalogMatching`**, **`DiveMediaFishialIdentificationStorage`**, **`FishialIdentificationReviewPresentation`**. Revisit privacy copy and App Store disclosures before shipping identify broadly or enabling background/auto-identify.

## App shell and navigation

- **Sign in:** **`SignInView`** — **`WaterBubbleBackground`** with a light full-screen **`surface`** scrim, **`GoDiveLogoPin`** + three-line tagline, native **`SignInWithAppleButton(.continue)`** (scopes **`.fullName`**, **`.email`**); **`AccountSession`** persists **`UserProfile.displayName`** in SwiftData. Apple **`fullName`** (once per Apple ID) or cached name is applied via **`applyDisplayNameFromApple`**; otherwise the profile stays **Diver** with no sign-up name sheet. **First-time** Apple ID sign-up shows **`NewAccountWelcomeView`** (**`AppPage`** + **GoDive** **`AppHeader`**) explaining upcoming Contacts + Photos access; **Continue** runs **`AppOnboardingPermissions`** (system prompts while still undetermined). Returning sign-ins skip welcome and permissions. **Sign out** on **Profile** clears the session pointer (local dives remain for that Apple ID).
- **Root:** `GoDiveMVP/Views/Pages/ContentView.swift` — **`TabView(selection:)`** with iOS 18+ **`Tab(value:)`** (Home, Logbook, Field Guide, Explore). Re-tapping **Logbook**, **Field Guide**, or **Explore** (list mode) uses the system tab-bar scroll-to-top behavior (iOS 18+), plus **`RootTabBarReselectForwarder`** + **`listScrollToTopTrigger`** for custom list layouts (pops stack, dismisses search focus, scrolls to top). **`tabBarMinimizeBehavior(.onScrollDown)`** (iOS 26 Liquid Glass) only collapses the bar while scrolling — it is not the scroll-to-top API.
- **Orientation:** **`AppPortraitOrientationLockController`** defaults to **portrait only** app-wide; **`ViewSingleActivity`** alone calls **`diveActivityLandscapeOrientation()`** (reference-counted unlock) for map / tank / media landscape. **`GoDiveGoogleMapsAppDelegate`** returns **`supportedMask`** for the window.
- **Tabs:** Home (`LogOverviewView`), Logbook (`LogbookView`), Field Guide (`FieldGuideView` — taxonomy browse + species detail), Explore (`ExploreView`) — all under **`Views/Pages/`** unless noted.
- **Navigation:** Home (and Logbook / Field Guide per current wiring) use **`NavigationStack`** for pushes. Pushed flows that should hide the system tab bar use **`hidesBottomTabBarWhenPushed()`** (e.g. Profile, Settings, Activity upload, Trip Planner, single-dive screen).
- **Secondary destinations:** Pushed headerless flows use **`SecondaryDestinationBackButton`** and related chrome from **`Views/Components/SecondaryDestinationChrome.swift`**.

## Headers and page wrappers

## Data storage strategy

- **Single source of truth:** All durable dive, account, catalog, equipment, certification, trip, and field-guide data lives in **SwiftData** via one **`ModelContainer`** (**`AppSwiftDataSchema`**) in **`GoDiveMVPApp`** (default **on-disk** store). **`@Model`** types include **`UserProfile`**, **`DiveActivity`**, **`DiveActivityEquipmentList`**, **`DiveEquipmentEntry`**, **`DiveProfilePoint`**, **`DiveBuddy`**, **`DiveBuddyTag`**, **`DiveMediaPhoto`**, **`DiveSite`**, **`DiveTrip`**, **`DiveTripActivityLink`**, **`DiveTripBuddyLink`**, **`MarineLife`**, **`MarineLifeUserRecord`**, **`SightingInstance`**, **`EquipmentItem`**, **`Certification`**; there is **no second database** and **no archive of raw import file bytes** (**.fit** / **.uddf**) after import.
- **Trips:** **`DiveTrip`** rows are owned by **`UserProfile`**; planning fields are dates, countries, optional title, optional catalog **`plannedSites`**, and optional planned roster buddies via **`DiveTripBuddyLink`**. **Profile → Trips** opens **`TripPlannerView`** (**`AppPage`**, title **Trips**) listing saved trips in **Upcoming**, **Active**, and **Past** sections (**`TripPlannerPresentation.listSections`** — calendar-day lifecycle via **`DiveTripActivityLinking.hasStarted`** + trip end date; **`TripPlannerListRow`** compact tiles match logbook row sizing: title, accent linked dive count on its own line (active/past), secondary date range caption below, optional trailing media preview (first linked trip media — tap opens trip **media** pager)); **Upcoming** trips use the logbook **`LogbookUpcomingTripBannerView`** (**Trip on the horizon** shout-out tile with airplane icon and accent border); **+** presents **`TripAddSheetView`** (**medium** sheet — name, dates, countries only; saved sites added on trip detail). relationship-seeded dives on first frame; on-demand owner dive/trip fetches for logbook **#** + trips); scoped sighting fetches; lazy pager pages; bottom scroll fade via **`HomeSheetPanelBottomScrim`** per pager page (**`homeSheetPanelBottomScrollFade`**, **`AppOverviewSheetPanelBackground`** hue, local bottom anchor + overlay so content scrolls under the band; page dots above); **Dives together** / **Trips together** / **Your tagged photos** — scrollable 3-column still grid; tap opens full-screen media edge-to-edge (portrait top chrome inset via **`LinkedMediaFullscreenPresentation.topChromeInset`**; swipe left/right for next/previous, swipe up/down or **X** to dismiss; **View on dive** opens dive in nested **`fullScreenCover`**; **View on dive** upper-right, **# of #**, star lower-right; header hero independent), **Edit**, contact badge on avatar → system contact picker to link or change; name/photo refresh from Contacts on open when linked). **`DiveBuddyEditSheetView`** includes red **Delete buddy** (confirmation) — removes roster row and untags all dives via **`DiveBuddyDeletion`**. Optional **`contactsIdentifier`** + photo sync from Contacts (**`DiveBuddyContactLinking`**). After **FIT** / **UDDF** import, **`DiveBuddyContactAutoLink`** tries to attach Contacts when access is **authorized** or **limited**: buddies on that import with no link get a single unambiguous fuzzy name match (**`DiveBuddyNameMatching`**); ambiguous or already-linked contacts are skipped. Removing a tag from a dive does not delete the roster row. Names that fuzzy-match the signed-in diver's profile display name are **not** tagged (import, manual add, seed). **Import** (UDDF + FIT via **`DiveBuddyImportConsolidation`**) decodes name-only pending tags, detaches them before insert, then **`DiveBuddyCatalog.findOrCreate`** with a per-file roster cache: exact normalized name, fuzzy match (**`DiveBuddyNameMatching`**), or new person; one buddy per name per import batch when names align.
- **User preferences (`UserDefaults` / `AppUserSettings`):** **`AppUserSettings.registerDefaultValues()`** runs at launch so **`automaticallyRenumberDives`**, **`useImperialDisplayUnits`**, and **`autoUploadMediaToActivities`** default **on** for new users (**`register(defaults:)`** never overrides a saved choice). **`automaticallyRenumberDives`** can rewrite persisted **`diveNumber`** after import, seed, or delete. **`useImperialDisplayUnits`** only drives **`EnvironmentValues.diveDisplayUnitSystem`** and **`DiveQuantityFormatting`** for on-screen strings; SwiftData fields stay in canonical **m** / **°C** / **psi** (no unit migration on toggle). **`defaultTankSize`** (**`DefaultTankSize`**) sets rated cylinder **cu ft** + **aluminum** / **steel** for new imports and gas rows when a dive has no material / volume from file. **`autoUploadMediaToActivities`** scans the Photos library for capture times within each dive window (**`DiveActivityMediaAttachWindow`**: dive-local wall-clock start + duration + ±2 min padding; timezone from site IANA / offset / GPS / place name; PhotoKit fetch spans local calendar day(s)). Matching uses the Photos **`creationDate`** absolute instant (**`shouldAttachAsset(creationDate:diveLocalOffsetSeconds:)`**) — EXIF **`capturedAt`** is kept for display ordering only, never to reject a match (cameras without **`OffsetTimeOriginal`** are mis-zoned). For **video** assets it also accepts a creation instant that lands in-window once the **dive-local UTC offset** is removed, so GoPro/action-cam clips (QuickTime stores camera local time as UTC) attach to true-UTC watch dives; the PhotoKit fetch is padded a day on each side so those recovered assets near local midnight are still enumerated. Matched assets attach as **library references** (**`DiveActivityMediaStorage.addLibraryReference`** — only the **`PHAsset.localIdentifier`** is stored, no exported file / inline bytes), so attach is instant, adds no disk, and avoids iCloud export failures; rendering loads them on demand via **`DiveMediaReferenceLoader`**. **`DiveLibraryMediaAttachDebug`** (os.Logger) traces window + per-asset decisions. The Photos permission prompt is requested on **profile setup** (**`ProfileEditSheet`** save when auto-upload on) and when **enabling the toggle** (backfill requests access before the empty-log short-circuit), gated by **`shouldRequestPhotoAccessForAutoUpload`** / **`hasResolvedPhotoLibraryAuthorization`**.
- **Dives:** Each saved dive is a **`DiveActivity`** row owned by the signed-in **`UserProfile`** (**`DiveActivityOwnership`** on import / sign-in; legacy unowned rows are **claimed** on next sign-in). The **Logbook** tab lists dives for **`AccountSession.currentProfile`** only (newest first). **`DiveActivityDiveNumbering`** owns chained **file-import** **`diveNumber`** assignment (**`nextChainedDiveNumberForNewImport`**), **`backfillMissingDiveNumbers`** (skips **`diveNumberExplicitlyNone`**), **`renumberAllChronologically`** (full **1…n** by **`startTime`**, skips **`diveNumberExplicitlyNone`** / logbook **`-`**), **`applyAutomaticSequentialRenumberIfNeeded`** (calls **`renumberAllChronologically`** when **`AppUserSettings.automaticallyRenumberDives`** is **on** — after import, seed, and after delete via **`DiveActivityDeletion`**), and **`renumberDivesNewerThanDeleted`** (tail-only; also skips **`-`** dives; tests / internal). **`DiveActivityDeletion`** uses **`DiveBackgroundDeletionWorker`** (**`@ModelActor`**) only — no second delete on the UI context; Logbook optimistic row hide + **`FetchDescriptor`** reconcile; detach nullify inverses on tags / owner / site / equipment; **profile / buddies / media / sightings** cascade from parent delete; equipment batch-deletes; **`deferRenumber`** tail **#** on **`DivePostDeleteRenumberScheduler`** after the progress dialog.
- **Ingress vs storage:**  
  - **JSON fixtures (opt-in Debug):** Bundled **`MockData/*.json`** via **`MockDataSeeder`** when **`MockDataSeeding.isLaunchSeedingEnabled`** is **`true`** (off by default).  
  - **File import (Logbook → Add activity):** **`.fit`** — read via **`FitDiveFileImport.readFitFileData`**, **`FitDiveFileDecoder`**, **`importFitData`** → **`DiveFileImportOutcome`** (**`userMessage`** + **`primaryInsertedDiveId`**). **`.uddf`** — **`UddfDiveFileImport.readUddfFileData`**, **`UddfDiveFileDecoder.buildDiveActivities`** (may return **multiple** dives per file), **`importUddfData`** (same outcome type; multi-dive files set **`primaryInsertedDiveId`** to the **newest** inserted dive by **`startTime`**, **`id`** tie-break). Before insert, **`DiveActivityDuplicateMatcher`** blocks or skips rows that match the log (**same `sourceDiveId`**, or cross-format **fingerprint**: start ±2 min, max depth ±0.6 m, bottom/duration tolerance — e.g. same dive from **FIT** and **UDDF**). **Logbook** shows **Possible duplicate** on matching pairs. Both paths use security-scoped **`Data`**, then **`DiveActivityDiveNumbering.assignNextDiveNumberChainedAfterNewest`** per dive (insert **oldest-first** for multi-dive UDDF so chained **#** matches chronology); **`insert`**; **`DiveActivityEquipmentAssociation.applyAutoAdd`** links owner gear with **`autoAdd`** (not retired); **`save`**. When **`AppUserSettings.automaticallyRenumberDives`** is **on**, **`applyAutomaticSequentialRenumberIfNeeded`** runs so **`diveNumber`** is **1…n** by date. Only canonical fields are written; **source files are not retained**. **`rawImportVersion`** / **`sourceDiveId`** record provenance (**FIT:** **`FileIdMesg`**; **UDDF:** generator string + **`<dive id>`**). **FIT** also fills **`bottomTimeSeconds`**, **`surfaceIntervalSeconds`**, **`waterTempAvgCelsius`** / **`waterTempMaxCelsius`** / **`waterTempMinCelsius`** from **`DiveSummaryMesg`** + session/record temps (**`DiveImportWaterTemperatureSummary`**); **UDDF** sets **`bottomTimeSeconds`** from **`diveduration`**, **`waterTempMinCelsius`** from **`lowesttemperature`** + waypoint min. **`DiveProfilePoint`**: **FIT** maps **`ndl_time`**, **`time_to_surface`**, **`ascent_rate`** when present; **FIT** / **UDDF** may set **`tankPressurePSI`** when the source includes per-sample cylinder pressure. **FIT** tank summary / updates → dive-level **`tankPressure*PSI`**, **`tankVolumeDescription`** (L + **~ft³** from **`TankSummary.volume_used`**); **UDDF** **`tankdata`** / waypoint **`tankpressure`**. After decode, **`DiveSACRMVCalculation`** sets **`avgSAC`** (**psi/min**) and **`avgRMV`** (**L/min**) when pressures, time, and depth allow; **RMV** uses rated tank **L** ÷ working **psi** (UDDF) or **FIT** **`volume_used` ÷ time** when no rated size. **FIT** entry **`DiveCoordinate`** prefers **`SessionMesg`** **`start_position_*`** (then first **`RecordMesg`** fix); **FIT** rejects **>1** diving session or **>2** tank sensor ids (**`FitDecodeError`**).
- **Catalog:** **`DiveSite`** rows (fixture-seeded **`divesites_sample`** when launch seeding is on) with optional **`country`**, **`region`**, **`bodyOfWater`**. On import, **`DiveActivitySiteAssociation`** links by exact import **`siteName`** when present (never a different nearby catalog site); duplicate catalog names disambiguate by GPS within that name set only; GPS match runs only when the dive has no import site name; unmatched import names create a new **`DiveSite`** (single **FIT** / **UDDF** always; bulk UDDF when **Create dive sites** is on). Import GPS stays on **`entryCoordinate`**; map pin uses linked **`siteCoordinate`** when present. Unlinked dives with location hints get a map-tab prompt to **`DiveSiteAddSheet`** (Place + map/coordinates; **info** control after dismiss). Import **`locationName`** prefills **region** / **country** on the add/edit sheet (**`DiveImportedLocationParsing`**: before/after first comma).
- **Not implemented yet:** Storing original FIT or JSON attachments, CloudKit / multi-device sync, export, or merge/delete-duplicate UX beyond **import block** + **Possible duplicate** logbook hint (**`DiveActivityDuplicateMatcher`**).

## Dive file import (Garmin FIT + UDDF)

- **FIT — dependency:** Swift package **[FITSwiftSDK](https://github.com/garmin/fit-swift-sdk)** (v21.x+) on **`GoDiveMVP`**; see **External dependencies** for the full dependency table.
- **FIT — decode + map:** **`Data/Import/FitDiveFileDecoder.swift`** — **`Decoder`** + **`FitListener`**; **`SessionMesg`** with **`Sport.diving`** (exactly **one** diving session per file); **`DiveActivity`** (**`source: .garminMK3`**, **`rawImportVersion`**); **`DiveSummaryMesg`** (session-linked) for **`bottomTimeSeconds`** / **`surfaceIntervalSeconds`**; **`DiveImportWaterTemperatureSummary`** for **`waterTemp*Celsius`**; **`DiveProfilePoint`** from **`RecordMesg`** (depth, time, temp, NDL, time-to-surface, instantaneous **ascent rate** m/s, optional per-sample **`tankPressurePSI`** from **`TankUpdateMesg`** time-alignment, optional **`heartRateBPM`**, **`po2Bars`**, **`n2Load`**, **`cnsLoad`** when present in the file); **entry GPS** from **`SessionMesg.start_position_*`** when set, else first **`RecordMesg`** position (invalid semicircle **`0x7FFFFFFF`** skipped); **tank** dive-level fields from **`TankSummaryMesg`** / **`TankUpdateMesg`** via **`FitTankFieldImport`** (bar→psi; **≤2** tank sensors; **`>2`** sensors or **>1** diving session → **`FitDecodeError`**); **gas** from **`DiveGasMesg`** (**`oxygenMix`** %, **`gasType`** **Air** / **Nitrox** via **`DiveGasMixImport`**); **`sourceDiveId`** from **`FileIdMesg`**. **`FitDecodeError`** for invalid FIT.
- **UDDF — decode + map:** **`Data/Import/UddfDiveFileDecoder.swift`** — **`XMLParser`** for **`http://www.streit.cc/uddf/3.2/`** (e.g. **MacDive** export); **`DiveActivity`** (**`source: .macDive`**), optional **tank** fields from **`tankdata`** (Pa→psi, m³ volume string, optional **`tankmaterial`**), **gas** from **`gasdefinitions/mix`** (**`<o2>`** → **`oxygenMix`** %) via **`tankdata/link`**, **`bottomTimeSeconds`** from **`diveduration`**, **`waterTempMinCelsius`** from **`lowesttemperature`** + waypoint min, profile from **`samples/waypoint`** (including waypoint **`tankpressure`** → **`tankPressurePSI`**), buddies from **`diver/buddy`** + links, site from **`divesite/site`**; **`UddfDecodeError`** for invalid / empty / no dives. Unmapped UDDF fields are listed in **`cursor/todo.md`** (UDDF section).
- **UI:** **Logbook** **`plus`** → **`ActivityUploadView`** — a grouped, fixed-layout picker with two sections: **Import from a file** (*Garmin FIT* **`.fit`**, *UDDF* **`.uddf`** — one or many dives) and **Add it yourself** (*Manual entry*). UDDF options include **MacDive Import** — pushes **`MacDiveUddfImportGuideView`** on **`ActivityUploadRoute`** (**`AppPage`**, compact non-scrolling step pages); last page **Import MacDive Data** opens the system picker. All three share one card style (gradient icon badge, file-type pill, subtitle, chevron, press feedback) via **`addActivitySourceCard`**. **`DiveFileImportOutcome.didSucceed`** / **`DiveFileImportSuccess`**. **Import dialog** is a simplified scrim — a loading bar + a single milestone line (**`DiveImportMilestone`**: **Reading File** → **Creating Dive Logs** → **Adding Media**); **`DiveImportOverlayState`** is **`.importing(milestone:fraction:)`** / **`.hidden`** / **`.failed`**. UDDF advances the bar within **Creating Dive Logs** (dives persisted) and **Adding Media** (PhotoKit attach); FIT splits media off via **`FitDiveFileImport.persistImportedActivity(attachMedia:)`** so the same milestone shows. Multi-dive UDDF still ends with the **`UddfImportSummary`** completion alert. SwiftUI **`.fileImporter`** (do not clear **`isPresented`** in **`onDisappear`**); **`DiveFileImporterPresentation`** for UTTypes + picker cancel. **Bulk UDDF** imports **dives only** (activity + profile/tank/site/buddies from UDDF) — **not** MacDive media, equipment locker, certifications, or other export data yet. **Settings** — import hint + **Default tank** + renumber + units.
