## Stack and entry

- **SwiftUI** UI, **SwiftData** persistence, iOS **26** target.
- **Maps:** **MapKit** when no Google key is configured. **Google Maps SDK** when **`Config/GoogleMapsSecrets.plist`** contains a valid **`APIKey`** (**`GoDiveMapEngine`**); **Explore**, dive overview, and site coordinate picker switch automatically. Optional **`MapID`** for hybrid POI styling. Launch arg **`-GoDiveMapEngineGoogle`** still forces Google without secrets (tooling).
- **`GoDiveMVP/GoDiveMVPApp.swift`:** `@main`; **`AppNetworkConnectivityMonitor`** (**`NWPathMonitor`**) at launch — offline uses local PhotoKit previews only (no iCloud / full-res fetch); dive media shows **`OfflineMediaUnavailableIndicator`** instead of retry error chrome. Production SwiftData via **`AppModelContainer.loadProduction()`** — container I/O begins in **`GoDiveMVPApp.init`** (**`beginLoadingProductionIfNeeded`**) on a background thread, then **`.modelContainer`** attaches when ready. **`AppLaunchOverlay`** bridges **`LaunchScreen.storyboard`** while the store loads and during the fast local session restore (**`AppLaunchSessionRestorePresentation`**); Sign in with Apple credential validation + unowned dive/buddy claims defer to **`AppLaunchSessionValidation`** after the overlay dismisses (offline-first — network failure keeps the cached session). Home featured media warms on **`LogOverviewView`** (slide **0** first, slides **1…n** deferred). Carousel excludes library videos **> 30 s**. **`AccountSession`** + **`AppSessionRootView`** gate **`ContentView`** behind **Sign in with Apple** (**`SignInView`**). **Launch mock seeding is off** (**`MockDataSeeding.isLaunchSeedingEnabled`** default **`false`**). **Launch maintenance:** **`AppLaunchMaintenance`** — dive # backfill, buddy legacy migration, marine catalog seed on a background **`ModelContext`** (not on the main thread at attach). **MapKit warm-up:** **`MapKitWarmup`** runs once ~**400 ms** after root attach (off-screen **`MKMapView`** only — no launch **`Map`** in the tree) so the first **Explore** / dive map is less janky — skipped under UI tests. **Google Maps warm-up:** **`GoogleMapsWarmup`** (async hidden **hybrid** **`GMSMapView`**, yields between SDK init and layout) runs when **`AccountSession.showsMainAppShell`** for cold launch; post-sign-up celebration configures the SDK only (no hidden map) before shell prewarm so Explore does not spawn duplicate **`CCTClearcutUploader`** instances — not during logged-out onboarding or post-sign-up gates; **`GMSServices`** configures lazily on first map view, not in **`didFinishLaunching`**. **UI tests** (**`-GoDiveUITest`** / env **`GoDiveUITest=1`**): **`GoDiveUITestRootView`** only — no SwiftData init, no MapKit. **Release:** no mock seeding on launch (see **Mock data & testing**). **Snappy navigation:** agent rule **`.cursor/rules/swiftui-snappy-navigation.mdc`** — keep tab/push/sheet transitions off the main thread for heavy work; **`HomeOverviewRebuildScheduler`** coalesces Home aggregate rebuilds; foreground activation skips full rebuild when the carousel is already displayable (**`HomeReturnNavigationPresentation`**).

### Google Maps — auth model (no user OAuth today)

The **Maps SDK for iOS** authenticates with a **server-side API key** bundled in the app (see **Secrets / setup** above). It does **not** sign the user into a Google account or send dive log data to Google for basic tile + marker display.

**Not required for current spike:** Google **OAuth consent screen** or “share data with Google Maps” user flow.

**Revisit before adding:** Google **Places** (autocomplete / search), **Routes**, **Geocoding** as a billed Google REST API, **Google Sign-In**, or any API that reads a user’s Google account or sends user-generated dive content to Google backends — those may need a privacy-policy update, OAuth consent configuration, and/or additional API enablement. Track in **`cursor/todo.md`** → **Google Maps / Google Cloud**.

### Fishial.AI — optional cloud fish identification

GoDive depends on **[Fishial.AI](https://fishial.ai/)**’s hosted **Recognition API v2** for the manual **Identify fish** flow on dive media (pink→purple gradient **sparkles** beside **+** on the **Marine life** tag sheet leading toolbar). The feature is **off unless configured**: **`FishialSecretsBootstrap`** loads **`ClientID`** / **`ClientSecret`** from gitignored **`GoDiveMVP/Config/FishialSecrets.plist`** (free developer keys from [portal.fishial.ai](https://portal.fishial.ai)).

**Selection UX:** photos open the crop editor on a fast local preview (**`.fastFormat`**, **1024** edge); videos scrub on the lighter **`.homeCarousel`** stream with a live-updating **`AVAssetImageGenerator`** still preview (not keyframe-snapped **`AVPlayer`** seeks) that refreshes as the slider drags — requests are coalesced (**`FishialVideoScrubFrameRequestCoalescer`**: one decode at a time, newest fraction runs next) at a small **±0.05 s** tolerance so intermediate frames show instead of only the frame after the finger lifts — and duration from **`PHAsset`** metadata — full iCloud originals load only when the user taps **Identify** / **Continue**. **What leaves the device:** one **JPEG still** per identify (user-cropped full-quality photo export or cropped scrubbed video frame) via **`POST /v2/recognize`**, plus optional dive coordinates in the **`Fishial-Location-Lat-Lon`** header when the associated **`DiveActivity`** has a resolvable map coordinate (linked site, entry GPS, or catalog name match). **No automatic batching** — the user must tap **Identify** after framing the fish in the crop editor (videos: scrub → **Continue** → crop → **Identify**).

**What stays local:** PhotoKit originals, the full dive log, and Field Guide catalog data. Fishial returns species candidates only; GoDive **fuzzy-matches** those scientific names against the bundled **`MarineLife`** catalog (**`FishialMarineLifeCatalogMatching`**) and shows catalog thumbnails in the review sheet. When the user confirms a match (single result) or picks one from multiple, GoDive creates a **`SightingInstance`** tag on that catalog species (**`MarineLifeSightingRecorder.tagSpecies`**) and saves the catalog scientific name on **`DiveMediaPhoto.fishialConfirmedSpeciesName`**; the matching species oval at **medium** / **large** shows an in-chip **sparkles** badge.

**Operational notes:** requires network and a signed-in profile to save catalog tags; subject to Fishial account credits, API availability, and their [API terms / privacy policy](https://docs.fishial.ai/api). Client code: **`FishialAPIClient`**, **`FishialObservationLocation`**, **`DiveMediaFishialIdentification`**, **`FishialMarineLifeCatalogMatching`**, **`DiveMediaFishialIdentificationStorage`**, **`FishialIdentificationReviewPresentation`**. Revisit privacy copy and App Store disclosures before shipping identify broadly or enabling background/auto-identify.

## Mock data & testing

- **Policy:** Non-trivial **`GoDiveMVP/`** changes should add or update tests in **`GoDiveMVPTests`** or **`GoDiveMVPUITests`** (see **`.cursor/rules/code-changes-require-tests.mdc`** and **`cursor/rules.md`**). Concurrency: **`@MainActor`** for UI state only; heavy SwiftData / compute off-main (**`.cursor/rules/swift-mainactor-usage.mdc`**, **`swiftui-snappy-navigation.mdc`**).
- **Debug builds:** **`MockDataSeeder`** available but **disabled** at launch by default; set **`MockDataSeeding.isLaunchSeedingEnabled`** to load bundled JSON. **Release:** never seeds on launch.
- **Unit tests:** **`GoDiveMVPTests`** — Swift Testing, logic-focused: matchers, **`.fit`** / **`.uddf`** import decoders + outcomes, **`DiveActivityDuplicateMatcher`**, **`DiveSACRMVCalculation_*`**, depth-profile math, **`diveLocationMapPresentation_*`** (incl. **`adjustedMapCenter`** with top + bottom obstructions), **`diveMapCoordinateResolver_*`**, **`diveActivityOverviewDetent_*`**, **`diveActivityOverviewPanelMetrics_*`** (scroll snap, detent helpers), tank tab (**`diveActivityTankPanelSummary_*`**), tab icons, **`DiveQuantityFormatting`**, **`DiveActivityDeletion`**, **`AppUserSettings`**, **`FitTankFieldImport_*`**, etc. Value types used from nonisolated tests (**`FieldGuideCategorySummary`**, **`FishialRankedSpecies`**, **`DiveActivityMapOverviewStatIcon`**, etc.) are **top-level** with explicit **nonisolated** **`Equatable`** / **`Hashable`** where needed (Swift 6 + **`SWIFT_DEFAULT_ACTOR_ISOLATION = MainActor`**).
- **Agent run/test:** **`.cursor/rules/xcode-run-test.mdc`** — when asked to run or test the app, the agent runs **`xcodebuild clean`**, then **`clean build`** / **`clean build-for-testing`** (e.g. Dre's Phone or iPhone 17 simulator), and reports compiler errors/warnings before handing off.
- **UI tests:** **`GoDiveMVPUITests`** — **`XCUIApplication.goDiveForUITesting()`** launches **`GoDiveUITestRootView`** (stable AX shell, not production **`ContentView`**). **`testLaunch`**: **`GoDive.UITest.Root`**, Home tab, Profile. Scheme: test bundles **`parallelizable = "NO"`**; **`runsForEachTargetApplicationUIConfiguration = false`**.
- **Manual:** **⌘R** in Debug on simulator/device with fixtures; previews where present.
