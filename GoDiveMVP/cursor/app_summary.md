# GoDive MVP App Summary

High-level architecture and conventions. For detailed **change history**, see `cursor/change_log.md`. For **coding rules**, see `cursor/rules.md`.

## Stack and entry

- **SwiftUI** UI, **SwiftData** persistence, iOS **26** target.
- **`GoDiveMVP/GoDiveMVPApp.swift`:** `@main`; production SwiftData via **`AppModelContainer.loadProduction()`** (on-disk **`ModelContainer`** init on a background thread, then **`.modelContainer`** on the root — avoids main-thread I/O hang at launch). **`AppLaunchOverlay`** bridges the system **`LaunchScreen.storyboard`** only while **`AccountSession`** restores the local profile (**`AppSessionBootstrapPresentation`**); Home featured media warms on **`LogOverviewView`** (slide **0** first, slides **1…n** deferred). Carousel excludes library videos **> 30 s**. **`AccountSession`** + **`AppSessionRootView`** gate **`ContentView`** behind **Sign in with Apple** (**`SignInView`**). **Launch mock seeding is off** (**`MockDataSeeding.isLaunchSeedingEnabled`** default **`false`**). **MapKit warm-up:** **`MapKitWarmupView`** + **`MapKitWarmup`** run once at launch (hidden **1×1** **`Map`**, brief off-screen **`MKMapView`**) so the first **Explore** / dive map is less janky — skipped under UI tests. **UI tests** (**`-GoDiveUITest`** / env **`GoDiveUITest=1`**): **`GoDiveUITestRootView`** only — no SwiftData init, no MapKit. **Release:** no mock seeding on launch (see **Mock data & testing**).

## App shell and navigation

- **Sign in:** **`SignInView`** — **`WaterBubbleBackground`** with a light full-screen **`surface`** scrim, **`GoDiveLogoPin`** + three-line tagline, native **`SignInWithAppleButton(.continue)`** (scopes **`.fullName`**, **`.email`**); **`AccountSession`** persists **`UserProfile.displayName`** in SwiftData. Apple **`fullName`** (once per Apple ID) or cached name is applied via **`applyDisplayNameFromApple`**; otherwise the profile stays **Diver** with no sign-up name sheet. **First-time** Apple ID sign-up (new **`UserProfile`** row) runs **`AppOnboardingPermissions`** — system Contacts then Photos prompts while still undetermined (returning sign-ins skip). **Sign out** on **Profile** clears the session pointer (local dives remain for that Apple ID).
- **Root:** `GoDiveMVP/Views/Pages/ContentView.swift` — **`TabView(selection:)`** with iOS 18+ **`Tab(value:)`** (Home, Logbook, Field Guide, Explore). Re-tapping **Logbook**, **Field Guide**, or **Explore** (list mode) uses the system tab-bar scroll-to-top behavior (iOS 18+), plus **`RootTabBarReselectForwarder`** + **`listScrollToTopTrigger`** for custom list layouts (pops stack, dismisses search focus, scrolls to top). **`tabBarMinimizeBehavior(.onScrollDown)`** (iOS 26 Liquid Glass) only collapses the bar while scrolling — it is not the scroll-to-top API.
- **Tabs:** Home (`LogOverviewView`), Logbook (`LogbookView`), Field Guide (`FieldGuideView` — taxonomy browse + species detail), Explore (`ExploreView`) — all under **`Views/Pages/`** unless noted.
- **Navigation:** Home (and Logbook / Field Guide per current wiring) use **`NavigationStack`** for pushes. Pushed flows that should hide the system tab bar use **`hidesBottomTabBarWhenPushed()`** (e.g. Profile, Settings, Activity upload, Trip Planner, single-dive screen).
- **Secondary destinations:** Pushed headerless flows use **`SecondaryDestinationBackButton`** and related chrome from **`Views/Components/SecondaryDestinationChrome.swift`**.

## Headers and page wrappers

### Top chrome & header strategy

- **Visual:** **`AppStatusBarEdgeScrim`** ( **`AppHeader.swift`** ) — **narrow** fade at the **status bar** only (**`surfaceElevated` → `surfaceGradientTop` → clear**); **GoDive** / **+** / dive tab rows sit on the uninterrupted **`screenBackgroundGradient`** (no full-row chrome tint). **Reduce Transparency** uses a short solid **`surfaceElevated`** band.

## Theme and chrome

- **`Views/Components/AppTheme.swift`:** semantic colors (light/dark), spacing, typography tokens — prefer these over one-off RGB in views. **Full-screen backgrounds** use **`screenBackgroundGradient`** (**`surfaceGradientTop`** → **`surfaceGradientBottom`**). **Status-only** top fade lives on **`AppStatusBarEdgeScrim`** in **`AppHeader.swift`** (not a second full-row gradient token). **`AppTheme.Layout.appHeaderClearanceFallback`** seeds header / Logbook spacer height before **`AppHeaderMetrics.HeightKey`** first fires. **List search:** **`CatalogSearchField`** (logbook **Search Activities**, field guide **Search Marine Life**, explore **Search dive sites**) — oval **`Capsule`** + **`AppTheme.SearchField`** accent outline. **Settings → Auto-upload media to activities:** when on, PhotoKit scan attaches library photos/videos by capture time within each dive window after dive import (**`DiveLibraryMediaAutoAttach`**); turning the toggle on backfills existing dives with **`DiveLibraryMediaBackfillProgressOverlay`**. **Sheets:** **`AppTheme.Sheet`** corner radius, **`contentTopSpacing`** (24pt below grabber), and material opacity (**0.64** modals; **0.82** embedded dive overview on **map** / **tank**); **`appSheetPresentationChrome()`** / **`appSheetContentTopSpacing()`** on every **`.sheet`** (**`AppSheetPresentation.swift`**, **`.cursor/rules/swiftui-sheet-standard.mdc`**).

## Data storage strategy

- **Single source of truth:** All durable dive, account, catalog, equipment, certification, and field-guide data lives in **SwiftData** via one **`ModelContainer`** (**`AppSwiftDataSchema`**) in **`GoDiveMVPApp`** (default **on-disk** store). **`@Model`** types include **`UserProfile`**, **`DiveActivity`**, **`DiveActivityEquipmentList`**, **`DiveEquipmentEntry`**, **`DiveProfilePoint`**, **`DiveBuddy`**, **`DiveBuddyTag`**, **`DiveMediaPhoto`**, **`DiveSite`**, **`MarineLife`**, **`MarineLifeUserRecord`**, **`SightingInstance`**, **`EquipmentItem`**, **`Certification`**; there is **no second database** and **no archive of raw import file bytes** (**.fit** / **.uddf**) after import.
- **Dive buddies:** **`DiveBuddy`** is the reusable person; **`DiveBuddyTag`** links them to dives (**dives together** = participation count on that roster row). **Buddies** edit sheet on a dive lists the owner roster (tap to tag/untag with check + highlight); **+** opens **New buddy** (name + **Connect to Contact**); **Profile → Dive Buddies** lists the roster (**`DiveBuddiesListView`**) with header **+** for roster-only add (same **New buddy** sheet) and pushed **`ViewDiveBuddyDetails`** (shared dives, **Edit**, contact badge on avatar → system contact picker to link or change; name/photo refresh from Contacts on open when linked). Optional **`contactsIdentifier`** + photo sync from Contacts (**`DiveBuddyContactLinking`**). After **FIT** / **UDDF** import, **`DiveBuddyContactAutoLink`** tries to attach Contacts when access is **authorized** or **limited**: buddies on that import with no link get a single unambiguous fuzzy name match (**`DiveBuddyNameMatching`**); ambiguous or already-linked contacts are skipped. Removing a tag from a dive does not delete the roster row. Names that fuzzy-match the signed-in diver's profile display name are **not** tagged (import, manual add, seed). **Import** (UDDF + FIT via **`DiveBuddyImportConsolidation`**) decodes name-only pending tags, detaches them before insert, then **`DiveBuddyCatalog.findOrCreate`** with a per-file roster cache: exact normalized name, fuzzy match (**`DiveBuddyNameMatching`**), or new person; one buddy per name per import batch when names align.
- **User preferences (`UserDefaults` / `AppUserSettings`):** **`AppUserSettings.registerDefaultValues()`** runs at launch so **`automaticallyRenumberDives`**, **`useImperialDisplayUnits`**, and **`autoUploadMediaToActivities`** default **on** for new users (**`register(defaults:)`** never overrides a saved choice). **`automaticallyRenumberDives`** can rewrite persisted **`diveNumber`** after import, seed, or delete. **`useImperialDisplayUnits`** only drives **`EnvironmentValues.diveDisplayUnitSystem`** and **`DiveQuantityFormatting`** for on-screen strings; SwiftData fields stay in canonical **m** / **°C** / **psi** (no unit migration on toggle). **`defaultTankSize`** (**`DefaultTankSize`**) sets rated cylinder **cu ft** + **aluminum** / **steel** for new imports and gas rows when a dive has no material / volume from file. **`autoUploadMediaToActivities`** scans the Photos library for capture times within each dive window (**`DiveActivityMediaAttachWindow`**: dive-local wall-clock start + duration + ±2 min padding; timezone from site IANA / offset / GPS / place name; PhotoKit fetch spans local calendar day(s)). Matching uses the Photos **`creationDate`** absolute instant (**`shouldAttachAsset(creationDate:diveLocalOffsetSeconds:)`**) — EXIF **`capturedAt`** is kept for display ordering only, never to reject a match (cameras without **`OffsetTimeOriginal`** are mis-zoned). For **video** assets it also accepts a creation instant that lands in-window once the **dive-local UTC offset** is removed, so GoPro/action-cam clips (QuickTime stores camera local time as UTC) attach to true-UTC watch dives; the PhotoKit fetch is padded a day on each side so those recovered assets near local midnight are still enumerated. Matched assets attach as **library references** (**`DiveActivityMediaStorage.addLibraryReference`** — only the **`PHAsset.localIdentifier`** is stored, no exported file / inline bytes), so attach is instant, adds no disk, and avoids iCloud export failures; rendering loads them on demand via **`DiveMediaReferenceLoader`**. **`DiveLibraryMediaAttachDebug`** (os.Logger) traces window + per-asset decisions. The Photos permission prompt is requested on **profile setup** (**`ProfileEditSheet`** save when auto-upload on) and when **enabling the toggle** (backfill requests access before the empty-log short-circuit), gated by **`shouldRequestPhotoAccessForAutoUpload`** / **`hasResolvedPhotoLibraryAuthorization`**.
- **Dives:** Each saved dive is a **`DiveActivity`** row owned by the signed-in **`UserProfile`** (**`DiveActivityOwnership`** on import / sign-in; legacy unowned rows are **claimed** on next sign-in).   - **JSON fixtures (opt-in Debug):** Bundled **`MockData/*.json`** via **`MockDataSeeder`** when **`MockDataSeeding.isLaunchSeedingEnabled`** is **`true`** (off by default).

## Dive file import (Garmin FIT + UDDF)

- **FIT — dependency:** Swift package **[FITSwiftSDK](https://github.com/garmin/fit-swift-sdk)** (v21.x+) on **`GoDiveMVP`**; **`Package.resolved`** in **`swiftpm`** shareddata.
- **FIT — decode + map:** **`Data/Import/FitDiveFileDecoder.swift`** — **`Decoder`** + **`FitListener`**; **`SessionMesg`** with **`Sport.diving`** (exactly **one** diving session per file); **`DiveActivity`** (**`source: .garminMK3`**, **`rawImportVersion`**); **`DiveSummaryMesg`** (session-linked) for **`bottomTimeSeconds`** / **`surfaceIntervalSeconds`**; **`DiveImportWaterTemperatureSummary`** for **`waterTemp*Celsius`**; **`DiveProfilePoint`** from **`RecordMesg`** (depth, time, temp, NDL, time-to-surface, instantaneous **ascent rate** m/s, optional per-sample **`tankPressurePSI`** from **`TankUpdateMesg`** time-alignment, optional **`heartRateBPM`**, **`po2Bars`**, **`n2Load`**, **`cnsLoad`** when present in the file); **entry GPS** from **`SessionMesg.start_position_*`** when set, else first **`RecordMesg`** position (invalid semicircle **`0x7FFFFFFF`** skipped); **tank** dive-level fields from **`TankSummaryMesg`** / **`TankUpdateMesg`** via **`FitTankFieldImport`** (bar→psi; **≤2** tank sensors; **`>2`** sensors or **>1** diving session → **`FitDecodeError`**); **gas** from **`DiveGasMesg`** (**`oxygenMix`** %, **`gasType`** **Air** / **Nitrox** via **`DiveGasMixImport`**); **`sourceDiveId`** from **`FileIdMesg`**. **`FitDecodeError`** for invalid FIT.
- **UDDF — decode + map:** **`Data/Import/UddfDiveFileDecoder.swift`** — **`XMLParser`** for **`http://www.streit.cc/uddf/3.2/`** (e.g. **MacDive** export); **`DiveActivity`** (**`source: .macDive`**), optional **tank** fields from **`tankdata`** (Pa→psi, m³ volume string, optional **`tankmaterial`**), **gas** from **`gasdefinitions/mix`** (**`<o2>`** → **`oxygenMix`** %) via **`tankdata/link`**, **`bottomTimeSeconds`** from **`diveduration`**, **`waterTempMinCelsius`** from **`lowesttemperature`** + waypoint min, profile from **`samples/waypoint`** (including waypoint **`tankpressure`** → **`tankPressurePSI`**), buddies from **`diver/buddy`** + links, site from **`divesite/site`**; **`UddfDecodeError`** for invalid / empty / no dives. Unmapped UDDF fields are listed in **`cursor/todo.md`** (UDDF section).
- **UI:** **Logbook** **`plus`** → **`ActivityUploadView`** — a grouped, scrollable picker with two sections: **Import from a file** (*Garmin FIT* **`.fit`**, *UDDF* **`.uddf`** — one or many dives) and **Add it yourself** (*Manual entry*). All three share one card style (gradient icon badge, file-type pill, subtitle, chevron, press feedback) via **`addActivitySourceCard`**. **`DiveFileImportOutcome.didSucceed`** / **`DiveFileImportSuccess`**. **Import dialog** is a simplified scrim — a loading bar + a single milestone line (**`DiveImportMilestone`**: **Reading File** → **Creating Dive Logs** → **Adding Media**); **`DiveImportOverlayState`** is **`.importing(milestone:fraction:)`** / **`.hidden`** / **`.failed`**. UDDF advances the bar within **Creating Dive Logs** (dives persisted) and **Adding Media** (PhotoKit attach); FIT splits media off via **`FitDiveFileImport.persistImportedActivity(attachMedia:)`** so the same milestone shows. Multi-dive UDDF still ends with the **`UddfImportSummary`** completion alert. SwiftUI **`.fileImporter`** (do not clear **`isPresented`** in **`onDisappear`**); **`DiveFileImporterPresentation`** for UTTypes + picker cancel. **Bulk UDDF** imports **dives only** (activity + profile/tank/site/buddies from UDDF) — **not** MacDive media, equipment locker, certifications, or other export data yet. **Settings** — import hint + **Default tank** + renumber + units.
